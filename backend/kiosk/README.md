Kiosk is a lightweight vectorstore (at least for now)


It's not going to be efficient, fast, or maybe lightweight

the purpose of this project is to get into the crux of how vector stores works, and how to implement them

I would like to use this subproject inside my libgen project, to store documents with embeddings, so this project is going to use langchain for the auxilary types and functionality


with this project, I would like to learn how vectorstores store documents for efficient lookups
